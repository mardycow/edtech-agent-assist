# **Итерации и анализ ошибок**

## **1. Прототип API Gateway**

Для первого этапа анализа ошибок была сформирована выборка из 15 примеров. Малый объем выборки обусловлен использованием модели с малым числом параметров в текущем прототипе API (высокая задержка ответов). В связи с этим анализ проведен в рамках одной итерации, направленной на проверку логики работы системы и оценки возможностей в базовой конфигурации.

### **Анализ ошибок и выявление проблем**

1. Несмотря на отсутствие в промпте перечня конкретных категорий, соответсвтующих запросам (задавались только общие темы), модель продемонстровала высокую связность. Это подтверждает, что архитектура промпта позволяет достигать локальной согласованности даже на абстрактных инструкциях.

```json
{
  "id": 4,
  "input": {
    "user_query": "как поменять номер телефона в лк?",
    "expected_category": "тех_поддержка"
  },
  "output": {
    "category": "Техподдержка",
    "draft_answer": "Для изменения номера телефона на платформе ЛК, пожалуйста, обратитесь в службу технической поддержки или проверьте настройки приложения.",
    "confidence": 1.0
  }
}
```

```json
{
  "id": 2,
  "input": {
    "user_query": "ребенок не понял задание по математике!",
    "expected_category": "объяснить_материал_задание"
  },
  "output": {
    "category": "Обучение",
    "draft_answer": "Понимаю, что вы не понимаете задание по математике. Давайте разберем это вместе!",
    "confidence": 0.6
  }
}
```

2. Текущая модель показала чувствительность к структуре выходных данных: только после явного описания полей в системном промпте она начала стабильно возвращать JSON, хотя ранее игнорировала требования к формату.

3. Местами наблюдаются ошибки в определении категорий и не всегда понимает смысл (для непрямых вопросов), что связано с отсутствием специфической базы знаний и примеров для узких тем.

4. Уровень галлюцинаций минимален: даже без явного указания в промпте модель не на все примеры выдает абстрактные инструкции, а чаще ссылается на отсутствие у нее необходимой информации, что демонстрирует адекватную самооценку (параметр confidence, который в случаях затруднений снижается).

```json
{
  "id": 13,
  "input": {
    "user_query": "Сколько еще ждать поступления средств?",
    "expected_category": "возврат_средств"
  },
  "output": {
    "category": "Оплата",
    "draft_answer": "Я не могу точно сказать, сколько еще ждать поступления средств, так как у меня нет доступа к вашей конкретной платформе или системе.",
    "confidence": 0.5
  }
}
```

### **Зоны роста**

- **Использование базы знаний**: для генерации верифицированных ответов на основе регламентов платформы;
- **Работа с промптами**: для повышения точности классификации непрямых запросов и генерации в соответствии с контекстом;

- _Примечание: Низкая скорость обработки и периодические ошибки API Error обусловлены использованием простейшей базовой модели для проверки логики. **Очевидно**, что в дальнейшем для AI-системы требуется выбор более мощной модели и реализация отказоустойчивого API._

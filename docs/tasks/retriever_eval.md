# **Оценка качества ретривера**

## **Сценарий использования и выбор метрик**

Ритривер в системе используется для извлечения документов из корпоративной базы знаний на основе вопроса пользователя. На данный момент ретривер необходим только логике RAG (с генерацией ответов на основе LLM) для минимизации галлюцинаций и ускорения работы оператора.

Для оценки качества ретривера будут использоваться метрики **Precision@1**, **Precision@3**, **HitRate@3** и **MRR**

Выбор метрик обусловлен спецификой базы знаний: для узких запросов (например, по конкретному курсу) релевантным является один эталонный документ, в то время как для других (подбор курса, общие вопросы) в выдачу могут попадать несколько верных документов. В рамках baseline-версии глубина поиска - k=3. _Precision@1_ позволит оценить наличие приоритетного документа, минимизируя риск первичного искажения ответа. _Precision@3_ будет контролировать наличие шума в контекстном окне, предотвращая перегрузку LLM нерелевантными данными при расширенной выборке. _HitRate@3_ гарантирует наличие хотя бы одного верного источника для генерации черновика, а MRR покажет качество ранжирования, подтверждая, что наиболее точные данные занимают верхние позиции, что важно для корректного восприятия информации как моделью, так и оператором.

## **Формирование набора данных для оценки**

Тестовый датасет был сформирован комбинированным методом для достижения высокой точности разметки. В качестве запросов использовались реальные паттерны обращений в техническую поддержку. База знаний и соответствующие ей документы были подготовлены синтетически. На основе этих данных посредством ручной разметки был сформирован эталонный набор оценки Qrels с четким определение релевантных документов для каждого запроса.

_Примечание: структура бенчмарка для оценки соответствует примеру из лекции (папка `RAG/benchmark`), а значения метрик для текущей версии ретривера были сохранены в файл `metrics_log.jsonl`_

## **Анализ результатов и возможные улучшения**

Текущие показатели подтверждают высокую работоспособность системы: **HitRate@3 (0.93)** и **MRR (0.82)** указывают на то, что релевантный документ почти всегда попадает в выборку и чаще всего занимает первую позицию: **Precision@1 (0.73)**. Значение **Precision@3 (0.34)** при k=3 является ожидаемым для базы, где на один запрос чаще всего приходится один эталонный документ.

Однако детальный анализ выявил:

- Разрыв в оценках документов: между первым и вторым результатами крайне мал (в q3 разница всего 0.0001). Это свидетельствует о высокой семантической близости документов в небольшой базе знаний.

- Наблюдается частое повторение одних и тех же документов (например, chunk_24, 25, 27) в топах разных запросов. Это указывает на наличие в них общих формулировок, которые могут притягивать векторы независимо от конкретного интента.

В дальнейшем это добавит сложностей на этапе работы LLM-генератора, поэтому стоит рассмотреть различные подходы к улучшению качества ретривера: использование гибридный поиск для отделения релевантных документов от шумов или применение уже реализованного классификатора для фильтрации поиска по категориям для разделения типов документов и др.

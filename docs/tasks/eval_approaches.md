# **Подходы для оценки критериев**

<table>
    <tr>

    </tr>
    <tr>
        <td>Задержка</td>
        <td>Точная оценка (функциональная корректность)</td>
        <td>Критерий проверяется без двусмысленности (детерминированный параметр),
        не требующий человеческое интерпретации,
        может быть проверен напрямую (программный тайминг работы)</td>
    </tr>
    <tr>
        <td>Способность следовать инструкциям</td>
        <td>Субъективная оценка (человеческая оценка)</td>
        <td>Критерий не может быть проверен без двусмысленности. Только человек может оценить корректность черновика и готовность отправки пользователю</td>
    </tr>
    <tr>
        <td>Фактичеcкая согласованность</td>
        <td>Субъективная оценка (человеческая оценка)</td>
        <td>Критерий не может быть проверен без двусмысленности. Относительно верификации по ссылкам
        может требоваться и логический анализ соответсвия ответа источнику (не только семантический или лексический)</td>
    </tr>
    <tr>
        <td>Предметно-ориентированная способность</td>
        <td>Точная оценка (функциональная корректность)</td>
        <td>Критерий проверяется без двусмысленности. Оцениваются дискретные метки классов -> можно проверить напрямую</td>
    </tr>

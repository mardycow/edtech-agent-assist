# **Обоснование внедрения RAG**

## **Описание данных**

На текущем этапе проектирования AI-системы (как MVP) предполагается, что система опирается на заранее подготовленный текстовый корпус, без прямой обработки различных медиафайлов. **Таким образом система работает с**:

- контекстом пользователя (история текущего диалога);
- текстовым корпусом (вся база знаний платформы: внутренние регламенты, описания курсов, транскрипты видеоматериалов, FAQ база);
- метаданными (ссылки на первоисточники базы знаний, опционально).

**Объем данных** - средний, но предусматривающий накопление.
**Динамика обновления** - периодическая (по мере необходимости), подгрузка новых документов и транскриптов (только для базы знаний: данные о контексте пользователя меняется динамически в рамках текущего диалога).

## **Стратегия доступа к знаниям**

Выбрана стратегия RAG с использованием польностью подготовленной векторной базы знаний, поскольку:

1. **Качество**: должно быть высоким (модель генерирует ответ только на основе данных внутренней базы знаний, что и реализует верификацию в рамках платформы)
2. **Стоимость**: оптимально невысокая (поиск только по заранее готовым векторам данных)
3. **Скорость**: необходимо высокая, для работы в реальном времени (т.к. генерируется черновик ответа для оператора)

## **Ожидаемые улучшения и риски**

Применение RAG позволит обеспечить предсказуемость и масштабируемость системы, точность ссылок на верифицированную информацию, а также снизить нагрузку на операторов поддержки.

Ключевые риски:
-потеря актуальности данных (при отстутсвии автоматического обновления векторов в БД в соответсвии с изменениями на платформе, преимущественно для обучающих материалов);

- потеря визуального контекста (при неправильной транскрибации медиафайлов или невозможности сослаться на доп. графики и изображения в видео).

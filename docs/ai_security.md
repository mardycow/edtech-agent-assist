# **Защита от атак**

## **Возможные атаки**

Для реализуемой системы возможны различные типы атак: обход ограничений (Prompt Injection или Jailbreaking), извлечение системного промпта, тренировочных данных или данных из внутренней БД, которая используется ретривером.

Основная угроза - пропуск на этапе классификации небезопосного вопроса и отправка его компоненту-генератору для формирования черновика ответа для оператора, который, в свою очередь, может не заметить подвоха.

В данной AI-системе, несмотря на предусмотренную защиту с помощью Human-in-the-Loop и неявного использования фильтрации на входе, важно не забывать и про другие подходы. И в дальнейшем, вне зависимости от изменений архитектуры:

- ограничивать доступ к лишней информации и системным инструкциям для каждого отдельного логического модуля;
- валидировать данные внутри каждого модуля, не полагаясь на выходы других (отдавать приоритет внутренним инструкциям).

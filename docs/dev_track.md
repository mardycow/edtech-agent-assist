# **Разработка архитектуры решения**

## **Этап 1. Прототип API Gateway (Baseline - `v0`)**

### **Технологический стек**

- **FastAPI**: Фреймворк для реализации HTTP API
- **Pydantic**: Механизм структурированного вывода и валидация JSON
- **Ollama**: Локальный запуск LLM
- **Model**: `qwen3.5:0.8b`
- **OpenAI-совместимый API**: Единый интерфейс взаимодействия с моделью через библиотеку `openai`

Также для осуществления будущей оценки сисмемы была выполнена ее [декомпозиция](./system_decomposition.md), сформулированы [критерии](./system_eval_criteria.md) и выбраны [подходы](./eval_approaches.md) для оценки каждого из них.

### **Архитектурное решение**

**Логика**: Объединение классфикации намерения пользователя и генерации ответа в одном запросе к модели (один промпт).

**Результаты**: В ходе анализа ошибок [прототипа `v0`](./error_analysis.md#1-прототип-api-приложения) выявлена нестабильность классификации из-за отсутсвия контекста и низкая скорость генерации ответов.
